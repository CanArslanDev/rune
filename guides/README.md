# Rune Guides

Extended documentation for the `rune` package and its sibling bridges. The root `README.md` is the 5-minute tour; these guides go deeper for developers who need the full picture.

## Start here

| Guide | When to read |
|-------|-------------|
| [Getting started](getting-started.md) | You just added `rune` to your app. Walks through your first `RuneView`, data binding, and named events. |
| [Source syntax reference](source-syntax.md) | You want the full list of Dart expressions Rune understands: widgets, value constructors, constants, operators, control flow. |
| [Cookbook](cookbook.md) | You know the basics and want copy-paste recipes for common patterns: two-way binding, conditional rendering, reactive counters, percent-of-screen sizing. |

## Ecosystem

| Guide | What it covers |
|-------|----------------|
| [Bridges](bridges.md) | Setup and usage for the five sibling bridge packages: `rune_cupertino`, `rune_provider`, `rune_router`, `rune_responsive_sizer`, `rune_devtools_extension`. |
| [DevTools extension](devtools.md) | How to install the Flutter DevTools extension, what it shows, and how to read the payload in the wild. |

## Debugging and operations

| Guide | What it covers |
|-------|----------------|
| [Troubleshooting](troubleshooting.md) | Common errors (`UnregisteredBuilderException`, `ResolveException`, `BindingException`), the `RuneException` hierarchy, and how to read the caret-pointer diagnostic. |

| [Contributing](../CONTRIBUTING.md) | If you are writing a PR, extending the resolver, or shipping a new bridge. |

## Reference

The API reference is generated by `dart doc` from the source and hosted by pub.dev on each published release: <https://pub.dev/documentation/rune/latest>.

## Architecture

The root `README.md` has a condensed architecture diagram. For the full picture (layered import rules, resolver dispatch order, extension points), see `CONTRIBUTING.md` and the architecture test at `test/architecture/import_flow_test.dart`. The architecture test is the executable specification of how layers import from each other; if it passes, the layering is sound.
